# Shadow Glossary

**非同期ビザンチン耐障害性（Asynchronous Byzantine Fault Tolerance(aBFT)）：** aBFTは、非同期ネットワーク内の分散システムに耐障害性を提供するために使用される合意形成アルゴリズムです。ビザンチンフォールトトレランス（BFT）アルゴリズムをベースとしており、コンピュータのクラッシュ、完全なサイバー攻撃、悪意のある操作などの障害が発生した場合でも、適切に設定され、許容できるネットワーク条件があれば、最終的にその合意に達することを保証しています。aBFTは、悪意や欠陥のある行為者がネットワーク内のノードの3分の1以上を占めていても、耐えられるように設計されています。\
\
**ブロックチェーン(Blockchain):** ピアツーピア、分散型、不変のデジタル台帳で、暗号を使用して永続的にリンクされ、安全かつ効率的にデータを保存するために使用されます。\
\
**ビザンチン耐障害性（Byzantine Fault Tolerance）:** 分散コンピューティングシステムにおける耐障害性システムで、複数のレプリカを使用して合意を得ることで、故障したノードがあっても許容でき、エラーや悪意のある攻撃にもかかわらずシステムの合意を維持できます。\
\
**コンセンサス・アルゴリズム(Consensus Algorithm):** コンセンサスアルゴリズムとは、分散システム間で単一のデータ値に関する合意を達成する方法です。一般的にブロックチェーンネットワークで使用され、共通のデータ状態に到達し、ネットワーク参加者全体のコンセンサスを確保するために使用されます。分散システムの耐障害性を実現するために使用され、分散システムの構成要素の一部が故障しても稼働を維持できるようにします。\
\
**コンセンサス・プロトコル(Consensus Protocols):** コンセンサスプロトコルとは、システム内のデータの状態について、分散システム間で合意を得るために用いられるアルゴリズムです。コンセンサスプロトコルは、分散システムにおいて、データの状態についてノード間の不一致がないことを保証し、どのノードもデータについて異なる見解を持っていないことを保証するために不可欠です。このプロトコルは、データ状態に関する必要な合意を得るために、多数決などの投票形式や、proof of workなどの手法を用いるのが一般的です。\
\
**暗号ハッシュ(Cryptographic Hashing)：** 暗号ハッシュは、データを保護し、暗号化されたコードを通してその真正性を検証するために、データを固定長の文字列、すなわち「ハッシュ」に変換するために使用されるプロセスです。ハッシュは元のデータに戻すことができないため、データの完全性を確保するために広範囲に使用されます。暗号化ハッシュは、データの完全性の検証、データソースの認証、改ざんの防止などに利用されています。\
\
**暗号技術(Cryptography):** 通信、データ、システムを読み取り不可能な形式に変換して保護するために使用される技術の実践と研究。暗号はサイバーセキュリティの重要な要素であり、データの保護と機密性を提供します。\
\
**データキャッシング(Data Caching):** データキャッシングとは、頻繁にアクセスされるデータや計算されたデータをキャッシュに保存し、必要なときにそのデータに素早くアクセスできるようにするソフトウェア工学の手法です。データキャッシングは、データを一時的に保存し、要求されたときに素早く提供することで機能します。これにより、アプリケーションのパフォーマンスを大幅に向上させ、リクエストに応える時間を短縮することができます。\
\
**データ暗号化(Data Encryption):** 暗号化アルゴリズムを用いてデータを暗号化し、復号化キーがなければアクセスできないようにするプロセス。データ暗号化により、権限のないユーザーによる機密データの読み取りが困難になります。\
\
**データインテグリティ(Data Integrity):** デジタルデータの品質で、そのライフサイクル全体にわたって維持され、一貫性があり、正しいものであること。この用語は、データの検証やエラーの検出と修正などの特定のプロトコルによって確保されます。\
\
**データ分割(Data Partitioning):** 大きなデータや処理能力を、より小さく、より管理しやすい単位に分割するプロセスです。この処理により、データ処理の精度が上がり、結果が早く出るだけでなく、データの保存やアクセスも簡単にできるようになります。\
\
**データシャーディング(Data Sharding):** データシャーディングは、大規模なデータセットを複数のノードにまたがって保存されるより小さなサブセットに分割する分割技術です。分散型データベースのスケーラビリティ、可用性、耐障害性を向上させるために使用されます。レプリケーションと組み合わせることで、クエリの並列実行を可能にし、クエリの速度を向上させることができます。\
\
**分散型ストレージネットワーク(Decentralized Storage Network):** 分散型ストレージネットワークは、分散したノードを利用して、単一障害点に依存することなく、安全にデータを保存し、提供する分散型システムの一種です。スケーラビリティと冗長性を備え、従来の集中型システムよりも安全で信頼性の高いアクセスを可能にする、高度なデータ保存・共有の形態です。\
\
**委任型プルーフオブステーク(Delegated Proof-of-Stake (DPoS)):** Delegated Proof-of-Stake (DPoS) は、いくつかのブロックチェーン・ネットワークで使用されている合意形成メカニズムです。トークン保有者が、自分の代わりにトランザクションを検証し、新しいブロックを生成する「代理人」に投票することで機能します。代理人は、取引手数料と新しいブロックの報酬の一定割合で報酬を得ます。DPoSネットワークは通常、従来のPoWやPoSネットワークよりも高速で拡張性に優れています。\
\
**デジタル署名(Digital Signatures)：** デジタル署名は、デジタルメッセージや文書の信頼性を証明するための数学的スキームです。メッセージや文書の送信元を認証したり、メッセージや文書の完全性を提供するために使用されます。有効なデジタル署名は、メッセージや文書が既知の送信者によって作成または承認され、転送中に変更されていないと信じる根拠を受信者に与えます。\
\
**有向非循環グラフ(Directed Acyclic Graphs):** 有向非循環グラフ（DAG）とは、サイクルを形成しない有向辺を持つグラフの一種です。頂点（またはノード）と、ノードをつなぐエッジから構成されます。ノード間のエッジの方向は、あるノードから別のノードへの情報の流れを決定します。DAGは、キューやツリーなどのデータセットをモデル化し、コンピュータ工学の複雑なアルゴリズムやプロセスを表現するのに有用な構造です。\
\
**分散型コンピューティング(Distributed Computing):** メモリ、ハードディスク、プロセッサなど、さまざまな計算資源を、単一のシステムとして動作する異なるコンピュータに分割して使用するコンピューティングの一種。分散コンピューティングの利点である拡張性、負荷分散、待ち時間の短縮、回復力の向上が得られます。データセンターやクラウドコンピューティングで広く利用されています。\
\
**分散型コンセンサス(Distributed Consensus)：** ある問題の状態や操作の順序について、分散型システムが合意に至るプロセスです。これは、システム内の接続された各ノードからの通信、検証、合意によって達成されます。\
\
**分散型データベース(Distributed Database):** 分散型データベースとは、コンピュータネットワークに接続された複数の機器にデータの異なる部分を保存することで、複数のユーザーによるデータアクセスの効率化や、異なる場所間でのデータ転送の効率化を可能にするデータベースの一種です。\
\
**分散ファイルシステム(Distributed File System)：** 分散ファイルシステムとは、複数のコンピューターがリモートサーバーにあるファイルにアクセスし、共有することを可能にするファイルシステムです。複数のコンピュータで構成されるコンピューティングシステムが、データを保存・管理するために連携することを可能にします。このシステムは、複数のノードにまたがる大規模な分散型ファイルストレージプラットフォームと見なすことができます。データはコンピュータ間で複製されるため、あるコンピュータが故障しても、別のコンピュータがその代わりを務めることができ、高可用性、拡張性、耐障害性を提供するのに役立ちます。\
\
**分散型台帳技術(Distributed Ledger Technology):** データベース技術の一種で、継続的に増加する記録のリストを維持し、それぞれがブロックとして保存され、暗号化によって保護されています。ネットワーク上の複数のノードで共有されるデータベースで、ノード間のデジタル取引を記録し、厳重なセキュリティと改ざん防止メカニズムで保護されています。取引記録は常に更新され、容易にアクセスすることができます。\
\
**分散型台帳技術プラットフォーム(Distributed Ledger Technology Platforms):** 分散型台帳技術（DLT）プラットフォームは、ネットワーク内の広範なノード間で分散的にデータを安全に共有・保存できる安全なデジタルプラットフォームです。DLTプラットフォームは、改ざん防止、暗号化プロトコルを用いたセキュリティ、悪意ある行為に直面しても回復力を保つことができる分散型台帳ソリューションを提供します。また、DLTプラットフォームは、安全でタイムリーなデータ交換を可能にし、取引のスケーラビリティ、信頼性、透明性を提供します。\
\
**分散型ストレージサービス(Distributed Storage Services):** ユーザーが複数の物理的なストレージにまたがってデータを保存できるようにするサービスです。これにより、データの信頼性と可用性が向上し、分散ワークロードがこれを活用できるようになります。\
\
**分散ストレージシステム(Distributed Storage System):** 分散ストレージシステムは、信頼性が高く安全な方法で大量のデータを管理、保存、バックアップするために一緒に相互作用する一連のコンピュータシステムです。分散型ストレージシステムは、コンポーネントが単一障害点に対して脆弱ではないため、従来のストレージシステムよりも耐障害性に優れています。このため、分散型ストレージシステムは、単一の集中型システムよりも高いレベルのデータの耐久性と可用性を提供することができます。\
\
**分散型システム(Distributed Systems):**： 複数の独立したコンポーネントから構成されるコンピューティングシステムの一種で、互いに通信して動作を調整・同期させ、全体として統一されたシステムを作り上げることができます。ネットワークで接続された複数のコンピュータのリソースを最大限に活用するように設計されたソフトウェア・アーキテクチャの一種です。\
\
**分散システムアーキテクチャ(Distributed Systems Architecture):**： 分散システムとは、複数の独立したコンピュータが相互に接続され、その活動を調整し、ネットワークのリソースを共有するもので、通常はメッセージパッシングインターフェースやリモートプロシージャコールを介して通信を行います。分散システムアーキテクチャには、システムの回復力、パフォーマンス、スケーラビリティ、可用性、およびセキュリティを確保するための設計ガイドラインとアプローチが含まれます。\
\
**消去符号化ファイルストレージ(Erasure Coded File Storage):** 消去符号化ファイルストレージとは、データを分割して保存し、それぞれのデータを様々なエラー訂正アルゴリズムを用いて複数回符号化するアーカイブ戦略です。この冗長性により、重要なデータを永続的に保存する際に、信頼性を高めつつ保存コストを削減できるため、消去符号化は有用です。\
\
**耐障害性(Fault Tolerance)：**： 耐障害性とは、ハードウェアやソフトウェアのエラー、障害、さらにはコンポーネントやデータの損失があっても、システムが正常な動作を継続する能力のことです。フォールトトレラントシステムは、故障に強く、システムの一部または全部が故障した場合でも正常な動作を継続するように設計されています。\
\
**ゴシッププロトコル(Gossip Protocol):** 分散システムの各メンバーが1つまたは複数のノードに定期的にメッセージを送信し、そのノードが他のノードに同じメッセージを送信して、ネットワークの全メンバーが同じメッセージを受信するまでの分散アルゴリズムです。これにより、システムは中央サーバーを必要とせずに、システム内の他のすべてのノードとレコードを認識し続けることができます。\
\
**ハッシュグラフ(Hashgraph):** 仮想投票アルゴリズムを用いて、従来の分散型台帳よりも高速に合意形成を行う分散型台帳プラットフォームです。認証のための第三者を必要とせず、デジタルオブジェクトやデータの安全な転送を可能にします。そのコンセンサスアルゴリズムはゴシッププロトコルに基づいており、ユーザーノード同士がニュースやアップデートを共有することができます。\
\
**ハイパーレッジャーファブリック(Hyperledger Fabric):** Hyperledger Fabricは、分散型台帳を利用したアプリケーションを開発するための基盤を提供するオープンソースソフトウェアプロジェクトです。複数の事業者間で安全かつ許可制の取引を可能にし、参加者のエコシステムがデータや資産を安全に交換できるようにします。スマートコントラクト、デジタル資産管理、暗号化、IDサービスなどをサポートします。\
\
**不変性(Immutability):** オブジェクトが、何度操作や変更を加えても、時間の経過とともに変化しない能力のことです。また、不変性とは、オブジェクトが変化せず、そのオブジェクトに対するすべての操作は、元のオブジェクトを変更する代わりに新しいインスタンスを返すという性質でもあります。\
\
**相互運用性(Interoperability):** メーカーや技術仕様が異なっていても、システムまたはコンポーネントが一緒に動作し、データを交換する能力。\
\
**鍵管理(Key Management)：** 鍵管理とは、ネットワーク上の転送中および保管中のデータを保護するために使用される暗号鍵のセットを全体的に管理することです。適切な鍵管理の暗号化と管理は、アプリケーションとシステム内の安全な通信を保証し、機密情報を保護するためのベースラインを提供します。鍵管理者は、暗号鍵の保管、ローテーション、更新に責任を持ち、データが不正な開示、変更、包含、または損失に対して安全であることを保証しなければなりません。\
\
**マークルツリー(Merkle Tree)：** マークルツリーは、大規模なデータセットの整合性を検証するために使用される、ツリーベースのデータ構造です。データノードの親子関係で構成され、ツリーの各ブロックはすべての子ブロックのデータを取り込んでハッシュ化し、自分自身のハッシュを作成します。このプロセスを1つのブロックが残るまで繰り返すことで、階層的でハッシュ化された構造を作り出しています。\
\
**マルチノードクラスター(Multi-Node Clusters):** 複数のコンピュータを接続したノードで構成され、互いに連携して1つのシステムとして動作するコンピューティングアーキテクチャの一種。マルチノードクラスターでは、情報処理やストレージの容量を増やすことができ、信頼性や可用性も向上します。\
\
**ノード(Nodes)：** ノードは、リンクリストやツリーデータ構造などのデータ構造の基本単位です。ノードにはデータが格納され、他のノードにリンクすることもあります。ノードは、グラフ、リンクリスト、ツリー、スタック、キューを実装するために使用されます。また、コンピュータサイエンスでは、アルゴリズムやデータ構造を表現するのにも使われます。\
\
**オラクル(Oracles)：** オラクルとは、ユーザーにクエリの結果を提供するために設計されたシステムです。このシステムは、ユーザーが提供したデータに基づいて、正確で信頼できる結論をユーザーに提供するために、しばしば頼りにされます。オラクルには、ハードウェアからソフトウェアベースのシステムまで、いくつかの世代があり、それぞれ異なる機能を備えています。\
\
**オーケストレーション(Orchestration):** オーケストレーションとは、クラウドベースのコンピューティングサービスを管理・構成するために、ソフトウェアによる自動化を行うプロセスです。複数のクラウドプラットフォームにまたがるワークロードの管理と展開を自動化するために使用され、組織は効率とスケーラビリティを得ることができるようになります。\
\
**ピア・ツー・ピア・ネットワーキング(Peer-to-Peer Networking):**： ピアツーピアネットワーキングとは、ネットワークアーキテクチャモデルの一種で、コンピュータ（またはピアノード）が、各ノードがネットワーク内の他のノードのクライアントまたはサーバーとして動作できるように接続されています。つまり、ネットワークに接続されたピア間の通信を管理する中央のサーバーに依存しないのです。
\
**プルーフ・オブ・ステーク(Proof of Stake (PoS)):** ブロックチェーン技術におけるコンセンサスメカニズムで、各ノードが保有するコインの量に応じて、ノードが取引を検証し、新しいブロックを生成することを可能にします。Proof of Work（PoW）コンセンサスプロトコルに代わるものです。PoSでは、検証者はコインを賭け、ブロックの検証を行う前にブロックチェーン・プロトコルにコインを預けなければならなりません。検証者は、ブロックを作成することで報酬を受け取り、悪意のある行為にはペナルティを受けます。\
\
**プルーフ・オブ・ストレージ(Proof of Storage (PoSt)):** Proof of Storageは、分散ストレージネットワークにおけるデータの保存を証明するために使用されるコンセンサス暗号メカニズムです。このコンセンサスモデルでは、ランダムに選ばれたストレージマイナーのサブセットが、データを正しく保存していること、必要なノードがすべてオンラインであることの暗号的証明を定期的に提供する必要があります。PoStの目標は、保存されているデータの安全性を確保するとともに、分散型ストレージネットワークの安全性と透明性を高めることです。\
\
**プルーフ・オブ・ワーク(Proof of Work (PoW)):** 「ブロック」と呼ばれる一連のデータを、計算負荷の高い数学的問題を解くことで検証する合意形成アルゴリズム。ブロックチェーンでは、作成された各ブロックが有効なPoWを持っていないとブロックが認められないため、セキュリティ対策として利用されています。計算問題の難易度は、ブロックチェーンが成長するにつれて時間とともに難しくなり、参加者にチェーンを継続させるインセンティブを与えています。\
\
**公開分散台帳(Public Distributed Ledger)：**公開分散台帳とは、ネットワーク上で行われたすべての取引に関する情報を保持する分散型データベースで、管理・検証する中央当局を必要とせず、すべての参加者によって分散・管理されています。ネットワーク上のすべての参加者は、台帳に保存されたデータにアクセスし、閲覧し、検証することができます。\
\
**公開鍵基盤（Public Key Infrastructure(PKI）)：** ネットワーク上のユーザー、コンピュータ、組織に関連する公開暗号鍵を管理、配布、識別、認証するプロトコル、サービス、標準のセットです。PKIは安全な通信を可能にし、意図した受信者だけがメッセージを読むことができることを保証します。\
\
**クイックP2P(QuickP2P)**： ピアツーピア（P2P）コンピューティングに特化したネットワークプロトコルの一種で、複数のコンピュータ間でファイルやデータを迅速に交換することができます。QuickP2Pは、ファイルを小さなブロックに分割し、検索するユーザーが並行して別々に高速にダウンロードできるようにする仕組みです。\
\
**クオラム(Quorum)**： 分散システムにおいて、コンセンサスを得るために従事しなければならないノードの最小数。定数値は、システム内のノード数に基づいて設定され、システム内に存在する全ノードの半分以上でなければなりません。\
\
**レプリケーション(Replication):** フォールトトレラントなデータ保存やネットワーク運用を目的として、分散システムにおいてデータやオブジェクトの冗長なコピーを作成するプロセスです。\
\
**ルーティングプロトコル(Routing Protocols):** ネットワークやインターネットを介したコンピュータ間のトラフィックを指示するプロトコルであり、データパケットのルーティングパスを決定し、ルーター間で利用可能なルートに関する情報を交換します。ルーティングプロトコルは、ルーターが互いに通信してネットワーク情報を交換し、データトラフィックに最適な経路を設定する方法を定義します。\
\
**スケーラビリティ（Scalability）:** コンピューティング、メモリー、データストレージ、ネットワーク帯域幅、電力などのリソースを追加したときに、システムの性能や能力を向上させる能力です。スケーラビリティは、データ量や作業量、ユーザー数の増加に対応しなければならないシステムを開発する際に重要な考慮事項です。\
\
**セキュアメッセージング(Secure Messaging):** セキュア・メッセージングとは、2つ以上の当事者間で、意図した受信者だけがメッセージの内容にアクセスできることを保証する目的で、暗号化されたメッセージを送受信するプロセスのことです。暗号化処理には一般的に公開鍵と秘密鍵が使用され、ネットワーク上で送信されたメッセージを他人が傍受することはほぼ不可能であり、安全です。\
\
**セキュアマルチパーティコンピューティング(Secure Multi-Party Computation (MPC))：**　コンピュータセキュリティのフレームワークで、複数の当事者がそれぞれの出力以外を他の当事者に明かすことなく、プライベートな入力に対して共同で関数を計算することを可能にするものです。MPCは、暗号の安全性を利用して、計算におけるプライバシーと安全性を実現するものです。\
\
**セキュリティプロトコル(Security Protocols):** セキュリティプロトコルとは、データを保護し、機器間の安全な通信を可能にするために、コンピュータネットワークが作成する標準的な規則や規制のシステムです。認証、暗号化、機密性、データの完全性などによく使われます。\
\
**スマートコントラクト(Smart Contracts):** あらかじめ決められた条件を満たすと、2つ以上の当事者間で合意された内容を自律的に実行する自己実行型のプロトコル。第三者や仲介者を介さず、安全かつ信頼性の高い方法で、金銭、資産、株式、その他価値のあるものを交換するために使用されます。スマートコントラクトは、ブロックチェーンベースのアプリケーションで広く使用され、リスクを低減し、スピードと精度を向上させています。\
\
**ステートマシン(State Machines):** ステートマシンは、遷移する状態からなるシステムであり、各状態は前の状態と現在の入力によって決定されます。各状態には条件と出力が付属しており、前の状態と入力条件が現在の状態の条件に一致すると、1つの出力が生成されます。ステートマシンは、コンピュータ工学において、有限の自動化のためによく使われます。\
\
**同期方法(Synchronization Methods)：** 分散アプリケーションやシステムの異なる部分が、ある時点で共有された同じデータセットから作業していることを保証する方法です。これは、コンポーネント間で積極的にデータを送信するか、コンポーネントがデータを要求するのを受動的に待ってから送信することで実現できます。\
\
**同期ビザンチン耐障害性（Synchronous Byzantine Fault Tolerance (SBFT)）：** ブロックチェーンネットワークのコンセンサスアルゴリズムで、与えられた時間内にすべてのネットワークメッセージが交換され検証されるためには、各ノードが接続されアクティブであることが必要です。このアルゴリズムは、一部の参加者が危険であったり悪意があったりする場合でも、分散型ネットワークが合意に至るための方法を提供します。\
\
**タイムスタンプ(Time-Stamping):** レコードに保存されているすべてのイベントに、ユニークで正確な時間値を割り当てて、それらのイベントの時系列を定義する技術です。\
\
**仮想投票(Virtual Voting)**： 市民がインターネットを通じて、直接または投票ポータルを通じて投票できるデジタル投票システム。特にパンデミック時には、物理的な投票所に代わるものとして利用されてきました。また、選挙の正確性や安全性を確認するためにも利用されます。\
\
**Web 3.0:** Web 3.0のコンセプトは、最新世代のインターネットを指します。AIやブロックチェーンなどの分散型台帳技術に基づき、高度に分散化・自動化されたものです。データの保存、認証、ウェブ上のデバイス間のインタラクションなど、よりオープンで安全なインフラを実現することができます。前身であるWeb 2.0よりもはるかにスマートで高速、かつ効率的なものになる可能性を秘めています。
